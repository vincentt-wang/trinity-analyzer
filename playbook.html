AI Workflow 引擎

每一階段都用同一套多代理引擎跑。這就是我們做這個網站時實際在用的流程，可重複、可移轉到任何專案。

1 研究扇出平行 agent 讀檔／搜尋
→
2 合成規格匯整成可執行 spec
→
3 建置確定性產出 HTML/code
→
4 對抗式審查多 critic 挑問題
→
5 最佳化迴圈依檢討再修
→
6 驗證Puppeteer 實測
→
7 部署git push 上線

研究扇出

一次派多個高階 agent 平行做：挖既有 IP、查市場、查投資機構、設計評審。互不干擾，只回結論。

對抗式審查

用 3D／互動／品牌／gstack 四個視角的 critic 撕開作品，找出「為什麼看起來廉價」，產出可實作的修正清單。

最佳化迴圈

critique → 修 → 再驗，重複到過關。每輪都 puppeteer 量測、零 console error 才部署。

角色與位置

角色 IP 與生態系漏斗

事業不只靠產品，靠你在生態系裡的位置與被信任的角色。下面是 Trinity／Vincent 在漏斗中的定位。

角色 IP

方法論的擁有者：創新三元 55 維度是可署名的 IP。HITL 的那個人：Trinity Review with Vincent，真人拆解＝最高信任也＝最高品質的訓練標註。把 founder 打磨乾淨的人：被你評過的案子，是更乾淨的 deal。

不可妥協的紅線

只說「被工具評估、最佳化過」，不說「幫你引薦／保證募到」。中立報告層公開、E-config 層留內部。暖引薦未來才開（EvoScale／GVA／Bonhoe）。信任就是護城河。
