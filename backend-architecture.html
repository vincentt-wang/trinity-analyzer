後台架構・可移轉地基

一個地基，很多積木

整個後台是一塊地基，由很多可重組的積木拼成。積木之間只透過宣告好的介面對話，所以每一塊都能獨立蓋、獨立測、獨立換，整套還能移轉到下一個專案。下面這張圖會動，讓你看見一份 deck 從上傳到寄出報告，怎麼穿過每一塊積木。

地基哲學

為什麼拆成積木

四個好處

可最佳化：每塊獨立，換掉一塊不動其他。可客製：機構要不同行為，換對應積木。可除錯：壞在哪塊一目了然。可移轉：下個專案拿同一套積木重拼，這才是真正的回報。

唯一的紀律

積木只透過宣告好的介面（stud 契約）對話，不偷看彼此內部。守住這條，上面四個好處才成立；破壞它，地基就退化成一坨泥。

下面把 25 塊積木分成六層。最底的平台／Ops 層是底板（baseplate），上面的功能積木卡上去。你之前點名的積木大多是獨立的一塊：入口守門、身份、邏輯、連線、爬蟲、AI、檔案、記憶 DB、通知、版控、展示，再加上計費、可觀測。其中工具箱（共用函式）、排程（Cron）、部署（Wrangler／Secrets）是橫切的平台能力，歸在底板 Ops 層，不另立功能卡。

持續微調迴路

概念護城河變資料護城河

run→capture 輸出＋脈絡→收結果／回饋→對 golden set 評分→改 prompt（新版本）→holdout A／B→promote→（定期）蒸餾／微調

每次 run 存不可變輸入、各透鏡原始輸出、prompt_version、model、token。回饋來自 founder 評分、+30/+90 天 Cron 信（聯絡了誰、回了嗎、募到沒），以及最高品質的專家更正（HITL）。Golden set 種子來自 B2B/Review/*（阿基里斯）。prompts 是資料：每個 prompt 是一個 version row，勝出且無退步才翻 active，回滾就是翻回旗標。機構資料也自己調：配對回饋更新 responsiveness／fit_priors，不動 LLM 就改善排序。

隱私與安全

Deck 是機密 IP

承重，不可妥協

直傳 R2 以 token 隔離（Worker 不持有位元組，15 分鐘 URL，40MB 上限，無列表）。不可枚舉的報告連結。靜態加密＋ TLS；secrets 只在 Wrangler／Secrets Store。Claude API 採排除客戶資料訓練條款，只送透鏡需要的，不記錄完整 deck 文字。明確 consent；公布留存政策，Cron N 天後自動刪除，提供 /api/erase。機構只給公開聯絡資訊，E-config 層留內部。報告絕不宣稱某基金已承諾或預先審核。
